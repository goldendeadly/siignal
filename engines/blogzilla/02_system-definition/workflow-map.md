# Workflow Map

This document maps the high‑level stages of a Blogzilla run, showing how tasks flow through contexts. Each phase corresponds to a context folder under `03_contexts/` and a workspace under `04_workspaces/`.

## Phase 1 – Source Intake

* **Context:** `source-intake`
* **Inputs:** blog post, URL or idea; run brief; resonance mode; core intent; niche.
* **Outputs:** registered source file; completed run brief; resonance mode selection; core intent selection; run folder creation.
* **Key steps:** read source, assign resonance mode and intent, create run folder and run brief.

## Phase 2 – Context Analysis

* **Context:** `context-analysis`
* **Inputs:** run brief; source file.
* **Outputs:** tone analysis; audience intent profile; conversion goal; primary keyword; LSI keywords; long‑tail variants; related questions; canonical statement; suggested publication cadence and silo.
* **Key steps:** determine tone; identify audience intent and conversion goal; map keywords and questions; write canonical statement.

## Phase 3 – Core Generation

* **Context:** `core-generation`
* **Inputs:** source file; analysis output.
* **Outputs:** summary (2–3 sentences); meta description (150–160 characters); 5–10 highlights tagged by type; modular asset tags.
* **Key steps:** write summary; compose meta description; extract highlights; tag highlights as framework, principle, statistic, provocation or meta‑insight.

## Phase 4 – Platform Generation

* **Context:** `platform-generation`
* **Inputs:** source file; analysis output; core assets.
* **Outputs:** channel‑specific drafts for X/Twitter, LinkedIn, Instagram, YouTube, Quora, Facebook, Pinterest, Medium, Substack, Blogger and Dev.to; CTA and hashtag sets.
* **Key steps:** generate posts tailored to each platform’s length and tone; include canonical statements and CTA tiers; save drafts to the run workspace.

## Phase 5 – Amplification & Expansion

* **Context:** `amplification-expansion`
* **Inputs:** source file; generated outputs.
* **Outputs:** 5 new blog ideas; 3 collaboration/backlink opportunities; 1 lead magnet idea; 1 automation improvement.
* **Key steps:** review outputs; analyse high‑performing topic vectors; propose new topics, collaborations, backlinks and lead magnets; suggest one system improvement or automation.

## Phase 6 – Tracking & Memory

* **Context:** `tracking-memory`
* **Inputs:** outputs; platform metrics.
* **Outputs:** updated master tracker; analytics summary; lessons learned; reflection logs.
* **Key steps:** log each output into the tracker; update engagement metrics; calculate performance; record lessons; identify top‑performing platform and resonance mode; prepare feedback for next run.

## Phase 7 – Automation & Deployment

* **Context:** `automation-deployment`
* **Inputs:** approved outputs; scheduling tools; design guidelines.
* **Outputs:** publishing schedule; VA/Canva instructions; automation notes; ready‑to‑publish assets.
* **Key steps:** organise outputs by platform and delivery time; prepare scheduling and automation instructions; generate deployment sheet; coordinate with VA/design resources.

## Feedback Loop

After Phase 7, move completed assets from `04_workspaces` to `08_outputs/` and archive the run folder. The analytics and expansion outputs feed into the next run’s planning. Use the `tracker` and `expansion` files to inform which topics and platforms to prioritise and what adjustments to make in resonance mode, CTA tiering and automation.
