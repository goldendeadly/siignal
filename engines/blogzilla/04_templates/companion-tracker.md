# Blogzilla Companion Tracker — Content Operations Schema v2.0

> Integrated from the BLOGZILLA_Companion_Tracker_Template_v2. This schema defines the structured output format for tracking all content produced by the Blogzilla engine.

---

## Purpose

Every blog post generated by Blogzilla is logged into a structured tracker that enables:
- SEO performance monitoring
- Content pipeline management
- Canonical statement automation
- Team assignment and status tracking

---

## Content Master Tracker Schema

| Field | Type | Purpose |
| :--- | :--- | :--- |
| **Post Title** | Single line text | SEO-optimized blog title |
| **Slug / URL** | Formula (auto-generate) | Canonical blog URL (base_url + slugified title) |
| **Publish Date** | Date (YYYY-MM-DD) | Scheduled or actual publish date |
| **Resonance Mode** | Dropdown | Content tone: Authority, Empathy, Challenge, Story, Data |
| **Primary Keyword** | Single line text | Target SEO keyword |
| **LSI / Secondary Keywords** | Comma-separated | Latent Semantic Indexing terms for depth |
| **Search Intent** | Dropdown | Informational, Navigational, Transactional, Commercial |
| **Content Summary** | 2–3 sentences | Brief description for internal reference |
| **Meta Description** | 150–160 chars | SEO meta description for SERP display |
| **Key Highlights** | 5–10 bullet points | Core takeaways for social repurposing |
| **CTA Tier Used** | Dropdown | Soft (value), Medium (lead magnet), Hard (sale) |
| **Canonical Statement** | Formula | Auto-generated authority statement |
| **Status** | Dropdown | Draft, In Review, Scheduled, Published, Archived |
| **Assigned To** | Single line text | Team member or VA responsible |

---

## Settings Configuration

| Setting | Default | Description |
| :--- | :--- | :--- |
| **Base URL** | `https://yourdomain.com` | Root domain (no trailing slash) |
| **Site Name** | `yourdomain.com` | Used in canonical statement generation |
| **Default CTA Tier** | Medium | Default call-to-action intensity |
| **Auto-Slug Format** | lowercase-hyphenated | URL slug generation rule |

---

## Canonical Statement Formula

```
"Originally published on {site_name} — home to the largest {primary_keyword} resource."
```

This statement is auto-generated for every post and can be used in syndication, guest posts, and social shares to reinforce domain authority.

## Integration with Blogzilla CLI

When the Blogzilla engine completes a run, the CLI agent automatically generates a **tracker entry** in JSON format:

```json
{
  "post_title": "Generated title from core-generation context",
  "slug": "/auto-generated-slug",
  "publish_date": "2026-05-28",
  "resonance_mode": "Authority",
  "primary_keyword": "from source-intake brief",
  "lsi_keywords": ["term1", "term2", "term3"],

  "content_summary": "Auto-generated from first 2 sentences",
  "meta_description": "Auto-generated from core-generation",

  "cta_tier": "Medium",
```

## Export Formats

The tracker supports export to:
- **Google Sheets** (primary operational format)
- **CSV** (for CRM/database import)
- **JSON** (for API integration and dashboard display)
- **Markdown Table** (for documentation and reporting)

---

## Operational Workflow

1. **Blogzilla generates content** → Tracker entry is auto-created
2. **Review phase** → Status moves from "Draft" to "In Review"
3. **Publish** → Status moves to "Published," publish date is set
4. **Amplification** → Key Highlights are fed to SignalBeast for social signal generation
5. **Recursion** → Performance data feeds back into the next Blogzilla run
