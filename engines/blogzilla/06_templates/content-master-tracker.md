# Content Master Tracker Template

This template outlines the structure for a master tracker that logs metadata for each blog post and its associated outputs. Use a spreadsheet (e.g. Google Sheets, Airtable) with the following fields:

| Field Name | Type/Format | Description |
| --- | --- | --- |
| Post Title | Single line text | SEO‑optimised blog title |
| Slug/URL | Formula | Canonical blog URL or slug |
| Publish Date | Date | Actual or planned publish date |
| Resonance Mode | Single select | Authority, Emotional, Technical, Viral, Educational |
| Primary Keyword | Text | Core keyword focus |
| LSI/Secondary Keywords | Multi‑select | Supporting semantic keywords |
| Search Intent | Single select | Informational, Educational, Transactional, Philosophical |
| Content Summary | Long text | 2–3 sentence overview of the blog |
| Meta Description | Long text | 150–160 character SEO snippet |
| Key Highlights | Long text | 5–10 concise insights for reuse |
| CTA Tier Used | Single select | Soft, Engagement, Conversion |
| Canonical Statement | Formula | “Originally published on [site] — home to the largest [keyword] resource.” |
| Status | Select | Idea, Writing, Editing, Published, Distributed, Archived |
| Assigned To | Collaborator | Responsible team member or VA |

Create this tracker at the start of a project and link platform outputs and analytics to it. Use formulas to auto‑generate slugs/URLs and canonical statements.
