# Context Handoff Template

Use this template when passing a run from one context to the next. It ensures continuity and captures key information.

- **Run Folder:** `run‑YYYY‑MM‑DD‑[topic-slug]`
- **Current Context:** (e.g. Source Intake, Context Analysis, Core Generation)
- **Next Context:** (e.g. Context Analysis, Core Generation, Platform Generation)
- **Summary of Work Completed:** (brief description of what has been done in the current context)
- **Files Generated:** (list of files created in this context)
- **Key Decisions or Assumptions:** (notes on tone, keywords, target audience, resonance mode, CTA tier)
- **Outstanding Questions or TODOs:** (anything that needs to be addressed in the next context)

Fill out this handoff document and save it in the run folder whenever transitioning contexts. This helps maintain clarity and reduces rework.
