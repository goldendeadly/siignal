# Tracking & Memory Workflow

1. **Gather outputs** – list all content pieces created in the run, noting their filenames, platforms and statuses.
2. **Update content master tracker** – record each output’s metadata (topic, resonance mode, primary keyword, LSI keywords, CTA tier, status, assigned collaborator) using the template.
3. **Update platform outputs tracker** – for each piece, log the platform, output type, headline/hook, URL, hashtags/keywords, CTA, publish date and initial performance metrics (if available).
4. **Log analytics** – gather metrics from each platform (impressions, engagement rate, CTR, ROI if available) and update the analytics sheet. Calculate performance scores or resonance scores if part of the template.
5. **Reflect & record lessons** – write a short reflection noting which platforms or content types performed best, any surprising insights, and recommended changes for the next run. Save as `reflection-[topic]-YYYY-MM-DD.md` in the run folder.
6. **Identify top performer** – determine which output or platform achieved the highest resonance and note it for future emphasis.
7. **Save trackers** – commit updated tracker files into `08_outputs/trackers/` with appropriate naming and version numbers.
8. **Hand off** – route to `automation-deployment` if publishing or scheduling tasks remain.
