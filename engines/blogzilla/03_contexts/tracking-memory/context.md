# Tracking & Memory Context

## Purpose

Maintain a record of outputs, performance metrics and lessons learned from each run. This context ensures that the system has a memory of what worked and what didn’t, enabling informed decisions for future runs.

## Read Before Starting

- The tracker templates in `06_templates/content-master-tracker.md` and `06_templates/platform-outputs-template.md` (once created).
- Existing tracking files for the topic (if any) in `08_outputs/trackers/`.
- The current run’s outputs.

## Ignore

- Source material and core assets (already processed).
- Expansion suggestions until Phase 5 is complete.

## Expected Outputs

- Updated content master tracker for the current run, logging titles, keywords, resonance mode, CTA tier, status and other metadata.
- Platform outputs tracker detailing each piece of content, its platform, hashtags, CTA and publication status.
- Analytics summary capturing metrics like impressions, engagement rate, CTR and top‑performing platform.
- Reflection notes highlighting lessons learned and suggestions for improvement.
