# Automation & Deployment Workflow

1. **Review approved outputs** – gather all final drafts from the run and confirm they have been reviewed or edited as needed.
2. **Map outputs to schedule** – decide when and where each piece will be published. Create a timetable that aligns with the recommended cadence (e.g. daily, weekly) and platform best practices (e.g. posting times).
3. **Prepare VA/Canva guidance** – for visual content (carousels, reels, pins), create a simple brief outlining slide structure, branding elements, and CTA placement. Provide any template references.
4. **Queue posts via scheduling tools** – for platforms using automation (Buffer, Metricool, Hypefury), prepare the post text, link to visuals, hashtags and scheduling time. Document which posts will be automated.
5. **Create a deployment sheet** – compile all scheduling data into a single document (e.g. `deployment-[topic]-YYYY-MM-DD.md`) listing platform, post filename, publish time, CTA, and automation status.
6. **Log readiness** – mark each output as scheduled/published in the platform outputs tracker.
7. **Hand off** – return to `tracking-memory` after publishing to update performance metrics.
