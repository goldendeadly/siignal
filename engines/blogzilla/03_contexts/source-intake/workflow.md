# Source Intake Workflow

1. **Locate the source** – identify the blog post, URL or idea in `07_inputs/source-blogs/`, or a keyword/idea in `07_inputs/keywords/`.
2. **Read the source** – skim the content to understand the main topic and intent.
3. **Determine niche/domain** – categorise the topic to help with keyword mapping later.
4. **Select resonance mode** – choose Authority, Emotional, Technical, Viral or Educational based on the intended tone.
5. **Select core intent** – choose SEO dominance, lead generation, thought leadership, audience growth or other.
6. **Create a run brief** – summarise the source, resonance mode, core intent, niche and any additional constraints. Save it as `run-brief-[topic]-v1.md` in the run folder.
7. **Create a run folder** – generate a new directory named `run‑YYYY‑MM‑DD‑[topic-slug]/` inside `04_workspaces/source-intake/`.
8. **Log the source** – record any relevant metadata such as publication date, author, or link for future reference.
9. **Pass to next context** – route the run to `context-analysis` once the brief and folder are ready.
