# Meeting Report Prompts

## Thinking Prompt

> **You are generating the weekly reports for the realtor team.  Using the logging sheet and weekly intake settings, compute weekly totals (leads processed, booked/attended/qualified consults, disqualified, no‑shows, unknown attendance) and the buyer vs. seller split.  Identify priority leads, at‑risk leads and bottlenecks.  Suggest up to three changes for next week.  Populate both the client‑facing and operator‑facing report templates.  If caps/replacement apply, create a replacement queue.  Populate the execution plan template based on totals and meeting settings.  Save all reports in the outputs folder.**

## File Editing Prompt

> **Open the weekly report templates (`weekly-report-template-client.md` and `weekly-report-template-operator.md`).  Create new report files using the naming convention.  Fill out each section with the computed data:**
> 
> - Team name, reporting week, time zone, service area, primary focus, report owner.
> - Definitions and rule citations (list the qualified and disqualify rules selected in the weekly intake).
> - Totals and buyer vs. seller split.
> - Priority leads (name, phone, intent, area, timeline, temperature, booking status, next action, notes).
> - At‑risk leads (name, phone, missing info or problem, recommended action).
> - No‑show and attendance notes (counts and top drivers).
> - Bottlenecks (top objections and missing‑info patterns).
> - Next week focus (1–3 changes).
> - For operator report: HOT, Warm, No‑Show and Missing‑Info queues; missing info requests; replacement summary; week plan (top five actions).
> - If cap model: fill in the replacement queue template with eligible, ineligible and pending records; fill in cap and replacement usage.
> - Populate the execution plan template with counts for daily blocks.
> 
> **Save the filled reports, replacement queue and execution plan in `08_outputs/weekly-reports/`.**

## Folder Processing Prompt

> **At the end of the reporting period, read the logging sheet and weekly intake file.  Compute metrics and populate the templates accordingly.  Save the client and operator reports, replacement queue (if applicable) and execution plan.  Archive the previous week’s reports to `09_archive/` if necessary.  Notify the optimisation context that the reports are ready.**
