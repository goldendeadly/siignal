# Performance Tracking Workflow

This workflow outlines how to log and analyse ad performance during the campaign.

## Steps

1. **Prepare tracker files**
   - Copy the templates from `06_templates/` into your run workspace:
     - `content-master-tracker.md` (rename to `tracker-[product]-campaign-v1.md`).
     - `platform-outputs-tracker.md` (rename to `tracker-[product]-ads-v1.md`).
   - Create a blank reflection document (`performance-reflections-v1.md`).

2. **Log initial run information**
   - Fill in the master tracker with run details: date, product, resonance mode, objective, budget and selected platforms.
   - Link to the offer details and messaging assets for reference.

3. **Record ad performance**
   - After each ad has been running long enough to generate data, collect metrics from the advertising platform’s dashboard:
     - Impressions
     - Clicks
     - Click‑through rate (CTR)
     - Conversions or sales
     - Cost per click (CPC)
     - Cost per acquisition (CPA)
     - Return on ad spend (ROAS)
   - For each metric, record the values in `tracker-[product]-ads-v1.md` next to the corresponding ad.

4. **Verify canonical lines and messaging**
   - Review the ad files to ensure each includes the canonical brand line.
   - Confirm that messaging remains consistent across platforms.

5. **Calculate performance scores**
   - Use the defined formula or weighting (e.g. `(CTR×0.4)+(Conversion Rate×0.4)+(ROAS×0.2)`) to calculate a performance score for each ad.
   - Record the score in the tracker.

6. **Identify top and underperforming ads**
   - Identify ads with the highest performance scores; note the messaging elements, hooks or audiences that contributed to success.
   - Identify ads with low performance; note possible reasons (weak hook, poor targeting, high CPC).

7. **Document reflections**
   - Summarise key insights and lessons learned in `performance-reflections-v1.md`.
   - Include qualitative feedback from comments or reviews if available.
   - Highlight any anomalies or unexpected results.

8. **Hand off to Optimisation & Expansion**
   - After logging and reflecting on performance, proceed to the Optimisation & Expansion context to propose tests and improvements.

## Outputs

- Completed tracker files and a reflections document containing insights and lessons.
