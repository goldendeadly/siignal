# Ad Assembly Context

## Purpose

The **Ad Assembly** context converts the core messaging assets into fully formed ads for each chosen platform.  It adapts the tone, length, format and CTA to meet the conventions and best practices of Facebook, Instagram, Google Ads, LinkedIn, YouTube, TikTok and other channels.

## Read First

- Messaging assets (`value-propositions.md`, `benefits.md`, `hooks-taglines.md`).
- Market analysis for audience and competitive context.
- Offer details for any technical or feature information.
- Platform guidelines or SOPs in `01_source-material/` for character limits, headline structures, etc.

## Use This Context For

- Choosing which platforms to create ads for based on the run brief and audience.
- Writing headlines, body copy and calls to action tailored to each platform’s style and length constraints.
- Suggesting creative concepts (images or video ideas) that support the copy.
- Tagging each ad with its platform, status (draft/review/final) and version number.

## Ignore

- Targeting specifics or budget (handled in the Targeting Plan context).
- Performance tracking (handled later).

## Expected Outputs

- A set of ad files saved in `04_workspaces/ad-assembly/[run-folder]/`, named according to the convention `[product]-[platform]-ad-[status]-v[number].md`.
- Each file should contain a headline, body copy, CTA and creative notes, along with the canonical brand line where applicable.
