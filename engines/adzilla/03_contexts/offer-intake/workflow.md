# Offer Intake Workflow

This workflow guides you through collecting and documenting the information required for a new AdZilla run.

## Steps

1. **Read the run brief**
   - Locate the run brief file in `07_inputs/run-briefs/` and open it.
   - Note the date, product name, resonance mode, marketing objective, selected platforms and budget/timeframe.

2. **Collect offer details**
   - Gather product or service descriptions from `07_inputs/source-blogs/`, `07_inputs/offer-files/` or other provided documents.
   - Extract key information: features, benefits, unique selling proposition, pricing, any promotions or bundles.

3. **Capture target audience persona**
   - Document demographic details (age, gender, location, income level, education, occupation).
   - Capture psychographic insights (interests, values, motivations, fears, objections).
   - Identify pain points and desired outcomes relevant to the offer.

4. **Record constraints and guidelines**
   - Note any regulatory or compliance restrictions (e.g. claims restrictions, industry‑specific rules).
   - Capture brand guidelines, style preferences or mandatory messaging (e.g. trademark usage, approved phrases).
   - Record budget and timeline constraints.

5. **Create the run workspace**
   - Create a folder named `run-[YYYY-MM-DD]-[product-slug]` under `04_workspaces/offer-intake/`.
   - Save an `offer-details.md` file inside this folder summarising all gathered information.

6. **Hand off to Market Analysis**
   - Once the intake is complete, proceed to the Market Analysis context to analyse the audience and market.

## Outputs

- A run folder containing `offer-details.md` with comprehensive intake information.
- Confirmation of resonance mode, marketing objective and platforms.
