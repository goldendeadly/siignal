# Desktop Prompts

Use the following prompts in a desktop environment (e.g. ChatGPT web or desktop app) when you need to plan or oversee a SignalBeast run.  These prompts are designed for high‑level thinking and coordination tasks.

## Run Planning Prompt

> **Objective:** Determine the overall plan for a new SignalBeast run.
>
> **Read:**
> - The run brief in `07_inputs/run-briefs/[run-file].md`.
> - The source material (blog post, keyword or idea) in `07_inputs/source-blogs/` or `07_inputs/keywords/`.
> - `03_contexts/context-decoder/context.md` to recall the purpose of the context decoding phase.
>
> **Tasks:**
> - Identify the resonance mode (Authority, Emotional, Technical, Viral, Educational or Hybrid) and the core intent (SEO dominance, lead generation, thought leadership, audience growth).
> - Confirm the niche/domain and any recursion amplification setting.
> - Decide which platforms should be activated in the platform explosion phase based on the target audience.
> - Outline the high‑level flow through all seven phases: context decoding, core generation, platform explosion, tracking & canon, signal expansion, automation suggestions and recursion engine.
>
> **Output:** A concise run plan summarising resonance mode, intent, selected platforms, and a phase‑by‑phase overview of the tasks ahead.

## Phase Coordination Prompt

> **Objective:** Coordinate progress across phases during a run.
>
> **Read:**
> - The current state of the workspace in `04_workspaces/`.
> - The run plan produced earlier.
>
> **Tasks:**
> - Determine which phase is currently active and what deliverables remain.
> - Identify dependencies (e.g. core assets must be complete before platform drafts begin).
> - Highlight any missing files or blocked tasks and suggest how to resolve them.
>
> **Output:** An updated status report indicating completed phases, outstanding tasks, and next actions for collaborators.
