# Automation Suggestions Context

## Purpose

The **Automation Suggestions** context focuses on identifying and outlining systems and tools to reduce manual workload.  It translates the process improvement idea from the Signal Expansion phase into concrete automation steps.  This corresponds to Phase 6 of the SignalBeast framework.

## Read First

- The `expansion-recommendations-v1.md` (or v2) file created in the Signal Expansion phase.
- Existing workflow diagrams or automation notes in `01_source-material/`.
- Any guidelines for automation platforms (e.g. Zapier, Make.com, Buffer, Metricool, Airtable).

## Use This Context For

- Designing a workflow that connects the SignalBeast content generation process with external tools such as spreadsheets, design platforms, schedulers and CRMs.
- Documenting triggers, actions and data flows for automations (e.g. auto‑creating tracker entries, scheduling posts, sending lead magnet emails).
- Specifying which tasks can be delegated to virtual assistants or automated scripts.
- Preparing VA instructions or SOPs for repeated tasks like carousel creation or scheduling.

## Ignore

- Directly generating new content or topics (handled in other contexts).

## Expected Outputs

- A workflow diagram or description outlining the automation flow (this may be written in Markdown or described textually).
- A VA instructions file if manual steps are still needed (e.g. creating visuals using Canva templates).
- A list of recommended tools or integrations with a brief description of how each will be used.
