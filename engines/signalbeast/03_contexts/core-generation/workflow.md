# Core Generation Workflow

This file describes the step‑by‑step process for the **Core Generation** phase of SignalBeast.  The goal is to create modular, evergreen assets that capture the essence of the source material and can be repurposed across platforms.

## Steps

1. **Review the context analysis**
   - Open `context-analysis-v1.md` from the current run workspace.
   - Note the tone, audience, intent, primary keyword and LSI/secondary keywords.

2. **Write the summary**
   - Craft a 2–3 sentence summary that captures the main idea of the source.
   - Ensure the summary reflects the selected resonance mode and intent.
   - Keep it high‑level: the summary should stand alone without requiring additional context.

3. **Write the meta description**
   - Compose a 150–160 character meta description for search previews and social snippets.
   - Integrate the primary keyword naturally and convey value to entice clicks.

4. **Extract key highlights**
   - Identify 5–10 of the most compelling insights, statistics or arguments from the source.
   - Each highlight should be self‑contained, concise and free of dependencies on surrounding text.
   - Write each highlight as a short statement or question that stands on its own.

5. **Tag each highlight**
   - Assign one of the following tags to every highlight:
     - **Framework** – outlines a structured process or model.
     - **Principle** – communicates a core idea or rule.
     - **Statistic** – presents numerical data or research findings.
     - **Provocation** – challenges assumptions or sparks debate.
     - **Meta‑Insight** – provides commentary on how to think about the topic.

6. **Save assets**
   - Create three files in the current run workspace under `core-generation/`:
     - `summary.md`
     - `meta-description.md`
     - `key-highlights.md`
   - Format `key-highlights.md` as a bullet list with the tag indicated in brackets after each highlight.

7. **Pass outputs to the Platform Explosion context**
   - Once complete, move on to the Platform Explosion phase to generate platform‑specific content using these assets.

## Outputs

- A set of core assets stored in the run folder: summary, meta description, and tagged key highlights.
