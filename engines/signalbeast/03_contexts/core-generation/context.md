# Core Generation Context

## Purpose

The **Core Generation** context is responsible for transforming the decoded analysis from Phase 1 into reusable content building blocks.  These core assets form the backbone of every platform‑specific output generated later.  In this stage you will create concise summaries, meta descriptions and a set of modular highlights that can be recombined across platforms.

## Read First

- The context analysis file (`context-analysis-v1.md`) produced in the Context Decoder phase.
- The original source blog or idea for additional nuance.

## Use This Context For

- Writing a succinct 2–3 sentence summary of the source material.
- Creating an SEO‑optimised meta description (150–160 characters) for previews and search snippets.
- Identifying 5–10 key highlights, statistics, or insights that can stand alone.
- Tagging each highlight with its type (Framework, Principle, Statistic, Provocation, Meta‑Insight).
- Preparing these assets for downstream platform conversion.

## Ignore

- Platform‑specific formatting rules (handled in the Platform Explosion context).
- Long‑form rewrites or expansions (handled later by the Recursion Engine).

## Expected Outputs

- `summary.md` – a concise summary capturing the core idea and tone.
- `meta-description.md` – a 150–160 character meta description incorporating the primary keyword.
- `key-highlights.md` – a list of 5–10 modular highlights with tags indicating whether they are frameworks, principles, statistics, provocations, or meta‑insights.
