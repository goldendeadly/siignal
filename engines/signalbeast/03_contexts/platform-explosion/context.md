# Platform Explosion Context

## Purpose

The **Platform Explosion** context takes the modular core assets and transforms them into a comprehensive suite of platform‑specific outputs.  It ensures that every major platform receives content in its native format, with appropriate tone, CTA layering, hashtags and canonical linkage.  This phase corresponds to Phase 3 of the SignalBeast system.

## Read First

- The core assets produced in the Core Generation phase (`summary.md`, `meta-description.md`, `key-highlights.md`).
- The context analysis file for reference on tone, audience, intent and keywords.
- Platform‑specific guidelines or SOPs (if provided) in `01_source-material/` or `02_system-definition/`.

## Use This Context For

- Generating Twitter/X threads, individual tweets and hashtags.
- Writing LinkedIn articles, posts and carousels.
- Designing Instagram carousel scripts and captions.
- Outlining YouTube long‑form video scripts and micro‑clip structures.
- Drafting Quora answers and Facebook captions.
- Creating Pinterest pin titles and descriptions.
- Adapting content for Medium, Substack, Blogger, Dev.to or Hashnode with canonical statements.

## Ignore

- Creation of additional topics or system automation (handled in later phases).
- Detailed design or visual creation (performed by VAs or design automation tools according to SOPs).

## Expected Outputs

- A set of Markdown or text files for each active platform within the run workspace, following naming conventions such as `topic-twitter-thread-v1.md`, `topic-linkedin-article-v1.md`, `topic-instagram-carousel1-v1.md`, etc.
- Each file should include the platform‑appropriate format, recommended hashtags or keywords, CTA tier (soft, engagement, conversion), and the canonical line.
