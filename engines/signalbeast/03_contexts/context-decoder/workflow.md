# Context Decoder Workflow

This file outlines the detailed process for the **Context Decoder** phase of the SignalBeast system.  In this context you analyse the raw input (a blog post, keyword, or idea) and extract all of the foundational metadata that drives every subsequent phase.

## Steps

1. **Read the source**
   - Open the blog post, keyword, or idea provided in the run brief.
   - Note the domain/niche and the user‑selected resonance mode and intent.

2. **Determine tone**
   - Identify whether the source reads as educational, emotional, technical, narrative, philosophical or viral.
   - Record the predominant tone as the baseline for later output adjustments.

3. **Identify the audience and intent**
   - Decide who the post is meant for (professionals, hobbyists, beginners, etc.).
   - Determine the conversion goal: awareness, authority, lead generation, retention or another clear call‑to‑action.

4. **Map the semantic SEO cluster**
   - Extract the **primary keyword** that the source should rank for.
   - List 3–5 **LSI/secondary keywords** and long‑tail variants.
   - Capture any related search questions or latent topics that should be addressed in subsequent outputs.

5. **Suggest publication cadence and silo**
   - Based on the topic and audience, recommend an initial publication frequency (e.g. weekly, bi‑weekly).
   - Suggest the broader content silo or category for this asset in the brand’s ecosystem.

6. **Create canonical consistency guidelines**
   - Draft the canonical statement to be used in every piece of derived content.  This includes the origin line (e.g. “This post originally appeared on…”), author tag, and backlink to the main article.

7. **Compile the analysis summary**
   - Summarise the above findings in a structured file (`context-analysis-v1.md`) in the corresponding run workspace.
   - Flag any assumptions or missing information for follow‑up in later phases.

## Outputs

- A structured analysis document (`context-analysis-v1.md`) containing tone, audience, intent, keyword map, long‑tail terms, related questions, cadence and silo suggestions, and canonical statement.

- A ready‑to‑use canonical line template for all subsequent outputs.
