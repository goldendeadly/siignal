# Tracking & Canon Workflow

This workflow outlines the procedures for logging outputs, verifying canonical statements and tracking performance in the **Tracking & Canon** phase.

## Steps

1. **Prepare tracking templates**
   - Copy the relevant templates from `06_templates/` into the run workspace:
     - `content-master-tracker.md` for a high‑level overview of the run.
     - `platform-outputs-tracker.md` for detailed platform output records.
     - `analytics-memory-template.md` for capturing performance metrics and reflections.

2. **Log outputs**
   - For each generated platform draft:
     - Fill in the platform name, output type (thread, article, carousel, script, etc.), headline/hook, hashtags/keywords, CTA tier and canonical link.
     - Record the planned publish date or actual publish date when known.
     - Note the resonance mode and run details.
   - Append the record to the **Platform Outputs** table.

3. **Update the content master record**
   - In the **Content Master Tracker**, create or update the entry for this run with the post title, slug/URL, publish date, resonance mode, primary keyword, LSI keywords, search intent, summary, meta description, key highlights, CTA tier used and canonical statement.

4. **Verify canonical statements**
   - Review each draft to ensure it contains the canonical line as defined in the context analysis.
   - Confirm the link is accurate and leads back to the source.

5. **Set statuses and ownership**
   - Assign a status (Idea, Draft, Review, Scheduled, Published, Archived) to each output.
   - Assign an owner or collaborator if necessary.

6. **Capture analytics post‑publication**
   - After publishing, collect engagement metrics: impressions, click‑through rate, engagement rate, open rate (for emails), etc.
   - Fill out the **Analytics & Memory Feedback** template with metrics and qualitative notes.
   - Calculate performance scores if formulas are provided.

7. **Reflect and document lessons learned**
   - Summarise what worked well, what underperformed and any surprising insights in the analytics template.
   - Identify top‑performing platforms and content types.
   - Note recommendations for future runs.

8. **Hand off to Signal Expansion**
   - Once tracking and reflection are complete, proceed to the Signal Expansion context to generate new ideas and backlinks.

## Outputs

- Updated tracker files containing all relevant run data.
- Verified canonical statements across all outputs.
- A reflection document capturing performance and lessons learned.
