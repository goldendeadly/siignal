# Tracking & Canon Context

## Purpose

The **Tracking & Canon** context handles logging, organising and reviewing all generated outputs.  It ensures canonical lines are applied consistently, CTAs are layered correctly, and platform metrics are captured for reflection.  This phase corresponds to Phase 4 of the SignalBeast system.

## Read First

- All platform drafts from the Platform Explosion phase.
- Any existing tracker templates or dashboards in `06_templates/`.
- The run brief to confirm objectives and resonance mode.

## Use This Context For

- Compiling a master record of every output, its platform, asset type, headline or hook, hashtags, CTA, canonical link and publication target.
- Checking that canonical lines are present and correctly formatted in each output.
- Assigning publish dates, statuses and owners to each asset.
- Updating analytics and performance metrics post‑publication.
- Capturing reflections and lessons learned for the Memory Log.

## Ignore

- Generation of new content (handled in previous phases).
- Suggesting new topics or automation improvements (handled in later phases).

## Expected Outputs

- An updated **Content Master Tracker** file with a record for the current run.
- A **Platform Outputs** tracking table with details for each generated piece.
- An **Analytics & Memory Feedback** document capturing metrics and reflections after publication.
- Verified canonical statements in all output files.
