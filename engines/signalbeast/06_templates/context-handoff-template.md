# Context Handoff Template

This template facilitates smooth transitions between contexts during a run.  Use it when one phase hands off to the next to ensure all necessary information is conveyed.

## From:

Specify the originating context (e.g. Context Decoder, Core Generation).

## To:

Specify the destination context (e.g. Platform Explosion, Tracking & Canon).

## Summary of Completed Work

Provide a brief overview of what has been accomplished in the current context, including key deliverables (e.g. summary, meta description, core highlights).

## Files to Read Next

List all files that the next context should read (e.g. `context-analysis-v1.md`, `summary.md`, `meta-description.md`, `key-highlights.md`, platform drafts).

## Notes and Considerations

Highlight any assumptions, outstanding questions or dependencies that the next context should be aware of.

## Next Steps

Provide a checklist of tasks the next context should complete, based on the workflow (e.g. generate platform drafts, populate trackers, design automation plan).
