# SignalBeast Content Engine

SignalBeast (codename: BLOGZILLA GIGA‑Prompt) is a recursive content deployment system. It takes a single blog post, keyword or idea and expands it into a 100‑piece content supergrid across all major platforms. The engine uses semantic signal encoding, platform‑aware CTA structuring, SEO cluster expansion and multi‑tone resonance modulation to propagate a “signal storm” that amplifies the original message.

## Purpose

* Transform one source asset into a self‑propagating network of platform‑native outputs.
* Generate a comprehensive SEO supercluster grid, canonical link mesh and publication matrix.
* Suggest new topics, collaborations, backlinks, lead magnets and automation improvements.
* Support recursive mutation via commands such as `/mutate`, `/threadcast`, `/glyphseed`, `/storybind`, `/promptforge` and `/recur100`.

## Inputs

- **Source content:** full blog post, idea or keyword.
- **Resonance mode:** Authority, Emotional, Technical, Viral, Educational or Hybrid.
- **Core intent:** SEO dominance, lead generation, thought leadership, audience growth.
- **Niche/domain:** e.g. systems thinking, roofing, jewellery, somatic movement.
- **Recursion amplification:** YES/NO – allows secondary mutation of outputs into tertiary forms.

## Outputs

- **100+ content assets** grouped by platform and content type: threads, carousels, posts, scripts, digests, pins, emails, prompts and rewrites.
- **SEO supercluster grid** mapping primary, LSI, question‑based and backlink keywords.
- **Platform publication matrix** showing which formats go to which channels and with what cadence.
- **Recursion pathways table** outlining mutation commands and next steps.
- **New topic suggestions, collaboration/backlink opportunities, lead magnet ideas and automation improvements.**
- **Self‑mutation feedback loop** with suggestions for enhancing the engine itself.

## Constraints and Rules

- Include a canonical line in every output: “This content originally appeared on [Your Site Name] — home to the largest [keyword] library on the web.”
- Apply CTA tiering (Soft, Engagement, Conversion) appropriate to each platform.
- Use SEO keyword layering (primary + 3–5 LSI terms) throughout the outputs.
- Adapt tone according to the selected resonance mode; support hybrid modes when specified.
- Maintain a tracking log for each output including platform, CTA, keyword usage, publication target and resonance score.
- Derive context names and file names from the workflow; do not invent arbitrary labels.

SignalBeast follows the same layered architecture as Blogzilla. See `00_router/Claude.md` for a detailed folder map. Context areas correspond to each phase of the SignalBeast workflow: context‑decoder, core‑generation, platform‑explosion, tracking‑canon, signal‑expansion, automation‑suggestions and recursion‑engine. Workspaces store active run files and outputs.
