# Context: Optimisation & Expansion

## Role
The recursive improvement layer that analyzes operational patterns, identifies inefficiencies, and recommends system upgrades. This context ensures Lead Shield gets smarter over time rather than stagnating.

## Scope
- Pattern analysis across all historical operational data
- Identification of new intent types or edge cases
- Script and prompt optimization recommendations
- Portfolio expansion opportunities (new properties, new services)

## Inputs
- QA Scorecards (historical)
- Weekly Operations Reports (historical)
- Classification failure logs
- Client feedback and escalation notes

## Outputs
- Optimization Report containing:
  - Pattern analysis (recurring issues, seasonal trends)
  - New intent recommendations (intents not currently classified)
  - Script updates (improved language for common scenarios)
  - Routing rule refinements
  - Expansion opportunities (additional services or properties)
  - ROI summary (time saved, leaks prevented, response improvement)

## Success Criteria
- Must produce at least 3 actionable recommendations per cycle
- Must quantify the impact of each recommendation (time saved or revenue protected)
- Must not recommend changes that would reduce QA pass rate
