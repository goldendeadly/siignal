# System Rules – Neural Marketing Engine v1

This document outlines the key rules and constraints governing the operation of the Neural Marketing Engine.  These rules ensure consistency, maintain quality and protect stakeholder interests.

1. **Preserve voice and tone** – All outputs must adhere to the conscious frequency specified in the run brief.  Do not switch tone mid‑flow unless directed by the optimisation phase.
2. **Respect avatar psychology** – Messaging must resonate with the identified avatar’s pains, desires and language.  Avoid generic copy that does not speak directly to the avatar.
3. **Maintain transformation focus** – Keep the primary outcome central across all assets.  Ensure the blog post, social signals and landing page consistently promise the same transformation.
4. **SEO alignment** – Use the primary keyword and semantic cluster provided in the strategy model.  Integrate them naturally without keyword stuffing.
5. **Call‑to‑action layering** – All social and conversion assets should include a CTA.  Layer CTAs (soft → engagement → conversion) as appropriate for the platform.
6. **Version control** – Never overwrite previous versions of assets.  If changes are made, increment the version number and update the file name accordingly.
7. **Context isolation** – Only read and write files within the current context and its workspace.  Do not edit other contexts’ files directly.
8. **Feedback loops** – Only enter optimisation when credible performance data or stakeholder feedback is available.  Uninformed changes can degrade performance.
9. **Archive responsibly** – Once a run is complete, archive its workspace and outputs to maintain a clean working environment.
10. **Confidentiality** – Treat all inputs and outputs as confidential marketing assets.  Do not share them outside of authorised channels.
