# Workflow Map – Neural Marketing Engine v1

The table below maps each phase of the engine to its inputs, core activities and outputs.  Use this as a high‑level guide to understand how information flows through the system.

| Phase / Context            | Inputs                                           | Activities / Steps                                                                                       | Outputs                                             |
|---------------------------|--------------------------------------------------|-----------------------------------------------------------------------------------------------------------|-----------------------------------------------------|
| **Input Intake**          | run brief template                               | Collect all required fields; normalise and validate the data; assign a run slug and folder name.          | Validated run brief stored in `04_workspaces/input-intake/`. |
| **Strategy Modelling**    | validated run brief; avatar inputs; offer inputs | Build an avatar map (demographics, psychographics, pains, desires); outline offer architecture (features, benefits, promise, pricing); select tone and conscious frequency; map primary outcome and transformation; compile a keyword cluster and search intent. | Strategy model document saved in `04_workspaces/strategy-modeling/`. |
| **Blog Generation**       | strategy model                                   | Generate seven blog title candidates; select or rank them; write a long‑form blog post based on the chosen title; incorporate SEO keywords and tone guidelines. | Title list and full blog post in `04_workspaces/blog-generation/`. |
| **Signal Expansion**      | strategy model; blog post                        | Decompose the blog into modular insights; craft platform‑specific posts (e.g. Twitter/X threads, LinkedIn articles/carousels, Instagram reels/carousels, YouTube scripts); add hooks, hashtags and CTAs. | Social signal set in `04_workspaces/signal-expansion/`. |
| **Conversion Assets**      | strategy model; blog post or offer model         | Write ad copy variations for each platform (e.g. search, social, display) using the offer architecture and tone guidelines; create a landing page draft (headline, sub‑head, problem, solution, benefits, social proof, CTA). | Ad copy set and landing page draft in `04_workspaces/conversion-assets/`. |
| **Optimisation & Recursion** | prior outputs; feedback data                    | Analyse performance metrics (CTR, engagement, conversions); identify weaknesses; adjust messaging, tone or structure; produce revised assets with incremented version numbers. | Revised outputs saved alongside previous versions in `04_workspaces/optimization-recursion/`. |
| **Packaging Output**        | all completed outputs                           | Compile an output index; copy final versions from workspaces into `08_outputs/final-delivery/`; compress or organise files as needed; prepare a summary and delivery notes. | Final delivery bundle and index in `08_outputs/final-delivery/`. |

> **Note:** Each phase builds upon the previous one.  Do not skip steps, and always save intermediate files in the appropriate workspace before moving to the next phase.
