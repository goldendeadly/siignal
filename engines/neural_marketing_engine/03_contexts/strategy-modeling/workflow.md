# Strategy Modeling Workflow

1. **Prepare**
   * Open the validated run brief from the current run folder.  Identify the niche, offer, avatar description, tone, outcome and SEO intent.
   * Gather any supplementary notes from `01_source-material/notes/` if provided.

2. **Avatar Mapping**
   * Use `06_templates/avatar-map-template.md` to create a detailed avatar profile.  Include demographics (age, gender, location), psychographics (values, interests, fears), pains, desires and objections.
   * Summarise the avatar’s current state versus desired state to contextualise the transformation.

3. **Offer Architecture**
   * Use `06_templates/offer-architecture-template.md` to outline the offer’s key features, benefits, unique selling propositions, pricing, guarantee and any bonuses.
   * Clarify how the offer solves the avatar’s pain points and facilitates the primary outcome.

4. **Tone & Conscious Frequency**
   * From the run brief, extract the chosen resonance mode (Authority, Emotional, Technical, Viral, Educational).  Define how this tone should manifest in copy (e.g. “confident and data‑driven” vs. “warm and story‑led”).
   * Note any language cues, vocabulary choices or stylistic requirements.

5. **Outcome Mapping**
   * Translate the primary outcome into key messages and sub‑goals.  Describe the transformation in terms that resonate with the avatar (e.g. “go from overwhelmed homeowner to empowered buyer”).
   * Identify supporting outcomes (secondary benefits) that can be woven into content.

6. **SEO Intent Mapping**
   * Use `06_templates/seo-intent-template.md` to specify the primary keyword, 3–5 LSI/secondary keywords and long‑tail queries.  Determine the search intent (informational, transactional, etc.).
   * Outline meta tags or on‑page elements that should be included in the blog post and landing page.

7. **Compile the Strategy Model**
   * Consolidate the avatar map, offer architecture, tone guidelines, outcome mapping and SEO intent into a single markdown file titled `[run-slug]-strategy-model-core-v1.md`.
   * Save it in `04_workspaces/strategy-modeling/`.

8. **Mark completion**
   * Once the strategy model is saved, note any open questions for the strategist (optional) and move to the `blog-generation` context.
