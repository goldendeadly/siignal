# Signal Expansion Workflow

1. **Review Source Materials**
   * Read the strategy model to recall the avatar, tone and outcome.
   * Read the blog post and highlight key sections, statistics, stories and takeaways that can be repurposed.

2. **Decompose the Blog**
   * Extract 5–10 modular insights or quotes from the blog.  Each should make sense on its own and align with the avatar’s pains or aspirations.
   * Tag each insight by type (e.g. framework, principle, statistic, story, provocation) to ensure varied content types.

3. **Generate Platform‑Specific Content**
   * For **Twitter/X**: Write a thread of 7–10 tweets.  Start with a bold hook, follow with succinct insights (one per tweet) and end with a CTA linking to the blog or offer.
   * For **LinkedIn**: Draft an article summarising the blog in 200–300 words with a professional tone and a concluding CTA.  Also design a carousel set: 5–7 slides, each containing one insight, a supporting graphic and a question or CTA.
   * For **Instagram**: Create carousel scripts and reel ideas.  Carousels should include a hook slide, 3–4 content slides and a CTA slide.  Reels should be 30–60 second scripts highlighting a single insight.
   * For **YouTube / TikTok**: Write short video scripts (30–90 seconds) and list visual cues or B‑roll suggestions.  Emphasise narrative flow to maintain viewer attention.
   * Add hashtags, emojis and formatting appropriate to each platform.  Use the primary keyword and 2–3 LSI terms as hashtags when possible.

4. **Assemble the Social Signal File**
   * Combine all platform outputs into a single markdown file named `[run-slug]-social-signal-expansion-v1.md`.
   * Use headings to separate platforms and bullet points or numbered lists to organise content.  Include notes for visuals or scheduling as necessary.

5. **Mark completion**
   * Save the social signal file in `04_workspaces/signal-expansion/` and proceed to the `conversion-assets` context.
