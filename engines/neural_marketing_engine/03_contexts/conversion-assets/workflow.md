# Conversion Assets Workflow

1. **Review Source Materials**
   * Read the strategy model to recall avatar pain points, offer benefits, tone guidelines and outcome mapping.
   * Scan the blog post for language that can be repurposed into ad headlines or body copy.
   * Open the conversion assets template in `06_templates/conversion-assets-template.md` to understand the required sections.

2. **Craft Ad Copy Sets**
   * Identify the advertising channels you will target (e.g. Google Search, Facebook Ads, LinkedIn Ads).
   * For each channel, write at least three ad variations:
     * **Headline** (max 30–60 characters depending on channel)
     * **Description** (100–180 characters)
     * **CTA** (e.g. “Learn More”, “Get the Guide”)
   * Tailor the language and format to each channel’s guidelines while maintaining the chosen tone and emphasising the primary outcome.
   * Include relevant keywords and emotional triggers.

3. **Draft the Landing Page**
   * Outline the page using these sections:
     1. **Hero Section** – headline reflecting the desired transformation and sub‑headline clarifying the promise.
     2. **Problem / Agitation** – describe the pain the avatar feels.
     3. **Solution / Promise** – present the offer as the solution, summarising key benefits.
     4. **Benefits List** – highlight specific benefits, using bullet points for clarity.
     5. **Social Proof / Credibility** – include testimonials, statistics or certifications if available.
     6. **Guarantee / Risk Reversal** – state any guarantees or risk‑free propositions.
     7. **CTA Stack** – present the call to action multiple times (e.g. mid‑page button, bottom form).
     8. **Form Instructions** – specify the fields required for the prospect to sign up or purchase.
   * Write the copy for each section, ensuring a logical flow and maintaining the conscious frequency.

4. **Compile Conversion Assets**
   * Save all ad copy sets in a file named `[run-slug]-ad-copy-conversion-v1.md`, organised by channel with headings.
   * Save the landing page draft as `[run-slug]-landing-page-conversion-v1.md`, structured with headings and subheadings.
   * Place both files in `04_workspaces/conversion-assets/`.

5. **Mark completion**
   * Once ads and landing page are drafted, proceed to the next context.  If feedback or optimisation is required, that will occur in the optimisation recursion phase.
