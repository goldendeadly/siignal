# Optimisation & Recursion Workflow

1. **Gather Feedback**
   * Locate feedback files in `07_inputs/feedback-inputs/`.  These may include metrics (CTR, engagement rate, conversion rate), audience comments and stakeholder notes.
   * Open the feedback loop template (`06_templates/feedback-loop-template.md`) to structure your analysis.

2. **Analyse Performance**
   * For each asset (blog post, social signal, ad copy, landing page), review the associated metrics and comments.
   * Identify which parts performed well and which underperformed (e.g. low headline click‑through, high bounce rate on landing page).

3. **Diagnose Issues**
   * Determine why underperforming elements missed the mark.  Did the tone misalign with the avatar?  Was the CTA unclear?  Did the keywords fail to match search intent?
   * Use the strategy model as a reference to ensure modifications remain aligned with the original plan.

4. **Plan Revisions**
   * Decide which assets require changes.  Prioritise high‑impact elements (e.g. headlines, introductions, CTA placement).
   * Create a revision list with specific actions for each file.

5. **Implement Revisions**
   * Open the original files in their respective workspaces (`04_workspaces/[context]/`).
   * Edit the content according to the revision plan.  Maintain the conscious frequency and avatar alignment.
   * Save the revised files with an incremented version number (e.g. `v2`).
   * Update the naming in the file header or metadata if necessary.

6. **Document the Changes**
   * Summarise what was changed and why in a `feedback-summary.md` file.  Include metrics for reference.
   * Save this summary in `04_workspaces/optimization-recursion/`.

7. **Mark completion**
   * After revisions are saved, notify the packaging context to use the updated versions.  Archive older versions if they are no longer needed.
