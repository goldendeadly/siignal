# Optimisation & Recursion Context

## Purpose

Refine existing marketing assets based on performance feedback and stakeholder comments.  This context enables iterative improvements, ensuring that the outputs become more effective over time.  Each pass produces incremented versions of the affected files.

## Read First

* Feedback files in `07_inputs/feedback-inputs/` (e.g. click‑through rates, engagement metrics, comments).
* Previous outputs in `08_outputs/` and their versions in the workspaces.
* The feedback loop template (`06_templates/feedback-loop-template.md`).

## Use This Context For

* Analysing performance data and qualitative feedback.
* Identifying weaknesses in messaging, tone, structure or CTA placement.
* Revising the blog post, social signal, ad copy and landing page to address these weaknesses.
* Incrementing the version numbers of revised files.

## Ignore

* Do not create entirely new campaigns or offers.  Optimisation is applied only to the current run.

## Expected Outputs

* Revised versions of any affected assets, saved with incremented version numbers (e.g. `v2` or `v3`) in the appropriate workspaces.
* A feedback summary outlining the changes made and the rationale, saved in `04_workspaces/optimization-recursion/`.
