# Packaging Output Context

## Purpose

Assemble all completed marketing assets into a final delivery package.  This context collates the latest versions of each output, produces an index for quick reference and ensures that files are properly named and placed in the final delivery folder.

## Read First

* The final versions of all outputs in `04_workspaces/` (blog assets, social signal, conversion assets, revised assets).
* The output index template in `06_templates/output-index-template.md`.

## Use This Context For

* Collecting final versions of: strategy model, blog titles list, blog post, social signal file, ad copy set, landing page draft and any revised assets.
* Copying these files to `08_outputs/final-delivery/`.
* Creating a consolidated output index file listing each deliverable, its version, file name and location.

## Ignore

* Do not generate new content here.  All assets should already be completed.
* Do not edit the content of the files; this context is only for organisation and delivery.

## Expected Outputs

* Final deliverable files in `08_outputs/final-delivery/`.
* An index file named `[run-slug]-output-index-final-v1.md` summarising the outputs and their locations.
